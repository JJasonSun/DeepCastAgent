# DeepCast 前端设计文档

## 概述

本文档记录了 DeepCast 前端的设计理念、组件架构以及所有优化改动的详细说明。整体设计遵循 **macOS 现代设计语言**，强调简洁、优雅、流畅的用户体验。

---

## 设计理念

### 核心风格：macOS 设计语言

- **毛玻璃效果 (Glassmorphism)**：使用 `backdrop-blur` 和半透明背景创造深度感
- **圆润的边角**：统一使用 `rounded-2xl`、`rounded-3xl` 等大圆角
- **微妙的阴影**：多层阴影营造悬浮感，如 `shadow-xl shadow-purple-500/10`
- **渐变色彩**：紫色系主色调，搭配粉色和蓝色的渐变过渡
- **流畅动画**：所有状态变化都有平滑过渡，使用 `cubic-bezier` 缓动函数

### 色彩体系

```css
/* 主色调 */
--primary: purple-500 / purple-600
--accent: pink-500 / fuchsia-500

/* 背景色 */
--bg-dark: #0f0f23 / slate-900
--bg-card: rgba(255, 255, 255, 0.03)

/* 状态色 */
--success: emerald-400 / green-400
--warning: yellow-400 / amber-400
--error: red-400 / rose-400
```

### 动画时间规范

| 元素类型 | 持续时间 | 缓动函数 |
|---------|---------|---------|
| 微交互 (hover) | 0.2-0.3s | ease |
| 状态切换 | 0.5s | cubic-bezier(0.4, 0, 0.2, 1) |
| 进度条 | 0.8-1.2s | cubic-bezier(0.4, 0, 0.2, 1) |
| 弹出动画 | 0.3-0.5s | cubic-bezier(0.34, 1.56, 0.64, 1) |

## 组件详解

### 1. SetupView.vue - 设置页面

**路径**: `frontend/src/components/SetupView.vue`

#### 设计特点

- **浮动光球背景**：三个不同大小的渐变光球，使用 `animate-float` 动画缓慢漂浮
- **品牌图标**：渐变色播客图标，带有光晕效果
- **输入框样式**：深色玻璃质感背景，聚焦时边框渐变发光
- **功能徽章**：展示核心功能的标签组，带有淡入动画
- **提交按钮**：渐变背景 + 光泽扫过动画

#### 核心 CSS 类

```css
.floating-orb {
  position: absolute;
  border-radius: 50%;
  filter: blur(60px);
  animation: float 20s ease-in-out infinite;
}

.input-glow:focus {
  box-shadow: 0 0 0 2px rgba(168, 85, 247, 0.3),
              0 0 20px rgba(168, 85, 247, 0.1);
}
```

### 2. ProductionView.vue - 制作流程页面

**路径**: `frontend/src/components/ProductionView.vue`

#### 设计特点

- **流程管道 (Pipeline)**：可视化展示制作进度的五个步骤
- **步骤指示器**：圆形图标，带有完成/进行中/待处理三种状态
- **连接线动画**：步骤间的连接线从灰色渐变填充为紫色
- **进度条**：顶部显示总体进度百分比
- **状态芯片**：显示当前阶段状态，支持取消状态

#### 流程步骤

| 步骤 | 图标 | 权重 |
|------|------|------|
| 深度研究 | 🔍 | 0-40% |
| 生成报告 | 📝 | 40% |
| 撰写脚本 | ✍️ | 55% |
| 语音合成 | 🎙️ | 70-95% |
| 成品合成 | ✨ | 95-100% |

#### 动画系统

```css
/* 步骤容器过渡 */
.pipeline-step {
  transition: background 0.5s cubic-bezier(0.4, 0, 0.2, 1),
              border-color 0.5s cubic-bezier(0.4, 0, 0.2, 1),
              opacity 0.5s ease,
              padding 0.3s ease,
              margin 0.3s ease,
              transform 0.5s cubic-bezier(0.4, 0, 0.2, 1);
}

/* 指示器弹出动画 */
@keyframes indicator-pop {
  0% { transform: scale(0.8); }
  50% { transform: scale(1.15); }
  100% { transform: scale(1); }
}

/* 连接线填充 */
.pipeline-connector::after {
  transition: height 0.8s cubic-bezier(0.4, 0, 0.2, 1);
}
.pipeline-connector.filled::after {
  height: 100%;
}
```

#### 状态类型

```typescript
type ProductionStage = "research" | "script" | "audio" | "done" | "cancelled" | "error"
```

### 5. App.vue - 主应用组件

**路径**: `frontend/src/App.vue`

#### 职责

- **视图路由**：管理 setup / production / player 三个视图的切换
- **状态管理**：集中管理生产状态和进度数据
- **事件处理**：处理后端 SSE 推送的各类事件
- **进度计算**：根据后端事件计算总体进度百分比

#### 进度计算逻辑

```typescript
// 进度权重配置
const progressWeights = {
  research: { base: 0, weight: 40 },  // 0-40%
  report: 40,                          // 40%
  script: 55,                          // 55%
  audio: { base: 70, weight: 25 },    // 70-95%
  synthesis: 95,                       // 95%
  done: 100                            // 100%
}

// 事件触发进度更新
handle_todo_list → 计算 research 阶段进度
handle_task_status → 更新任务完成数
handle_stage_change → 切换阶段基础进度
handle_audio_progress → 音频合成进度
```

#### SSE 事件类型

| 事件名 | 说明 |
|--------|------|
| `todo_list` | 研究任务列表 |
| `task_status` | 单个任务状态更新 |
| `stage_change` | 生产阶段切换 |
| `audio_progress` | 音频生成进度 |
| `research_result` | 研究结果 |
| `production_complete` | 生产完成 |
| `production_error` | 生产错误 |

## 功能性修复记录

### 1. 进度同步问题

**问题**：前端进度百分比与后端实际进度不同步

**解决方案**：
- 将进度计算逻辑从 `ProductionView.vue` 的 computed 移至 `App.vue` 的 reactive 状态
- 使用 `taskProgress = reactive({ completed: 0, total: 0 })` 跟踪任务完成情况
- 根据后端 SSE 事件实时更新进度

### 2. 进度显示问题

**问题**：进度百分比在模板中不显示

**解决方案**：
- 使用 `toRef(props, 'progressPercent')` 确保响应式传递
- 设置初始进度为 2%，避免显示 0%
- 重命名为 `progress` 提高代码可读性

### 3. 取消状态处理

**问题**：点击取消后显示 100% 和"制作完成"

**解决方案**：
- 新增 `'cancelled'` 状态到 `ProductionStage` 类型
- 取消处理器设置 `productionStage = "cancelled"`
- UI 根据 cancelled 状态显示"已取消"而非"制作完成"

### 4. 错误终态处理

**问题**：后端返回 `error` 或前端流式连接异常时，页面只写日志但流程卡仍停留在处理中，用户无法判断是否已经失败。

**解决方案**：
- 新增 `'error'` 状态到 `ProductionStage` 类型
- 后端 `error` 事件或前端请求异常时设置 `productionStage = "error"`
- UI 状态芯片显示"制作失败"，并保留日志中的具体错误信息

### 5. 步骤过渡动画

**问题**：步骤切换时动画生硬

**解决方案**：
- 为所有管道元素添加 0.5-1.2s 的 CSS 过渡
- 使用 `cubic-bezier(0.4, 0, 0.2, 1)` 缓动函数
- 新增 `indicator-pop`、`indicator-glow-in` 关键帧动画
- 连接线使用 `::after` 伪元素实现高度填充动画
