## 数据收集优化：从“搜索”到“研究”

将数据收集从简单的“关键词搜索”升级为智能化的“深度研究（Deep Research）”。

**核心思路：迭代式查询与信息饱和** 智能体不应只搜索一次。它应该像一个真正的研究员一样：

1. **初始查询** ：根据初始目标生成几个核心搜索查询。
2. **结果分析** ：分析返回的结果，提取关键实体、概念和新的问题。
3. **查询精炼与扩展** ：基于分析出的新信息，生成更具体、更深入的新查询。例如，如果初始搜索“A公司财报”，发现了“XX新兴业务”，那么下一步就应该立即搜索“A公司XX业务市场份额”、“XX业务主要竞争对手”等。
4. **循环与终止** ：重复步骤2和3，直到新的搜索不再返回有价值的新信息，达到“信息饱和”状态，或者达到了预设的搜索深度/次数限制。

## 内容生成优化：追求专业与深度

要生成一篇高质量的研报和播客，简单的分段生成是不够的。

可以进行迭代式精炼（Iterative Refinement）

1. **草稿生成** ：首先让模型根据大纲和数据生成一个章节的草稿。
2. **自我批判** ：然后，让一个“批判者”Agent（或者用一个带有批判性指令的 Prompt）来评估这份草稿。评估维度可以包括：逻辑是否严谨？数据支撑是否充分？语言是否专业？是否存在矛盾？
3. **迭代修改** ：将批判意见和原始草稿一起反馈给“作者”Agent，让它进行修改。这个过程可以重复多次，直到内容质量达标。
