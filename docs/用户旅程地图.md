# DeepCast 用户旅程地图

## 概述

用户从"产生知识需求"到"完成知识吸收"的完整旅程，分为 5 个阶段。

---

## 阶段 1：发现需求

**场景：** 用户在日常中产生知识需求——看到一篇行业新闻、同事提到一个概念、刷到一个话题。

**用户行为：** 意识到自己需要深入了解某个主题，但没有整块时间阅读。

**情绪状态：** 焦虑（"又一个不懂的概念"）、无奈（"没时间看"）

**痛点：**
- 知道需要学习，但找不到合适的时间和方式
- 搜索引擎给的结果碎片化，需要自己拼凑
- 看长文太累，看短视频太浅

**机会：** 在用户产生需求的瞬间，提供一个"零门槛深度了解"的选项。

---

## 阶段 2：输入主题

**场景：** 用户打开 DeepCast，在输入框中输入想要了解的主题。

**用户行为：** 输入一句话主题（如"AI Agent 的发展趋势"），点击"开始制作播客"。

**情绪状态：** 期待（"看看 AI 能给我什么"）、轻微怀疑（"真的能自动生成播客？"）

**痛点：**
- 不确定该输入什么才能得到好的结果
- 担心 AI 生成的内容太浅或不准确

**产品设计决策：**
- 极简输入：一个文本框 + 一个按钮，零配置门槛
- 示例提示：placeholder 显示"AI Agent 的发展趋势"引导输入
- 功能标签：底部展示"混合搜索引擎"、"深度 AI 研究"、"自然语音合成"设置预期

**机会：** 提供"灵感库"（预设热门话题 Tag），降低"不知道问什么"的认知负担。（v1.5 计划）

---

## 阶段 3：等待生成

**场景：** 用户提交主题后，等待系统完成研究和生成（约 2-3 分钟）。

**用户行为：** 观看进度条和 Terminal 日志，了解系统当前在做什么。

**情绪变化曲线：**
```
0-15s:  好奇 ("它在做什么？")
15-30s: 期待 ("快好了吗？")
30-60s: 焦虑 ("怎么还没好？")  ← 关键流失节点
60-90s: 安心 ("看到它在搜索和总结了")
90-120s: 兴奋 ("快完成了！")
```

**痛点：**
- AI 生成是"黑盒"，不知道在做什么、还要多久
- 等待时间长，容易关闭页面
- 担心中途出错白等

**产品设计决策：**
- **Terminal UI**：实时展示 Agent 动作——"正在搜索 Tavily..."、"正在总结任务 2/3..."、"正在生成对话脚本..."。将黑盒变为半透明。
- **进度条 2% 起步**：初版从 0% 起步，用户反馈"以为卡住了"。改为 2% 起步，消除静止错觉。
- **阶段可视化**：四步时间线（深度研究 → 剧本创作 → 音频合成 → 制作完成），每步有完成/进行中/待处理三种状态。
- **取消功能**：红色"取消"按钮 + 确认弹窗，给用户控制感。

**机会：**
- 在等待期间展示"趣味知识"或"相关话题推荐"，转移注意力
- 首段音频就绪后立即允许播放（流式播报），缩短感知等待时间

---

## 阶段 4：收听播客

**场景：** 播客生成完成，用户进入播放器页面。

**用户行为：** 播放音频，同时可浏览右侧的研究报告。可能下载 MP3。

**情绪状态：** 惊喜（"居然真的生成了"）、满足（"内容还不错"）

**产品设计决策：**
- **黑胶唱片播放器**：旋转的黑胶唱片 + "DeepCast Original" 标识，营造"收听播客"而非"播放文件"的仪式感。
- **双栏布局**：左侧播放器 + 右侧报告，支持"图文对照阅读"。
- **下载 MP3**：满足离线收听需求（通勤地铁无网络场景）。

**痛点：**
- 播放器缺少倍速控制（1.2x/1.5x）
- 无法跳转到特定对话段落
- 报告太长时滚动体验不佳

**机会：**
- 添加倍速播放（v1.5）
- 添加对话时间戳索引，点击跳转
- 添加"分享"功能，生成分享卡片

---

## 阶段 5：后续行为

**场景：** 用户听完播客后的行为。

**可能的行为路径：**

```
路径 A（满意）→ 下载 MP3 → 分享给朋友 → 回来制作新播客 → 转化为付费用户
路径 B（部分满意）→ 觉得内容不够深 → 希望能指定更多细节 → 等待 v2.0 审批功能
路径 C（不满意）→ 内容太浅或不准确 → 放弃使用 → 流失
```

**痛点：**
- 没有历史记录，之前生成的播客找不回来
- 无法对生成结果进行反馈（好评/差评）
- 想基于同一个主题深入研究，但需要重新输入

**机会：**
- 播客历史架（v1.5 计划）：记录所有生成过的播客，支持回听
- 反馈机制：每个播客旁加"有帮助/没帮助"按钮，收集质量数据
- 迭代入口："基于这个主题深入研究"按钮，复用已有搜索结果

---

## 关键体验指标对应

| 旅程阶段 | 关键指标 | 目标值 |
|---------|---------|--------|
| 输入 → 等待 | TTFA（首段音频时间） | < 30s |
| 等待 | 等待超 30s 留存率 | > 90% |
| 等待 → 播放 | 播放点击率 | > 80% |
| 收听 | 完播率（>80% 时长） | > 35% |
| 后续 | 7 日回访率 | > 20% |
| 后续 | 免费转付费率 | > 5% |

---

## 情绪曲线总览

```
情绪
  高 │          ●                              ●
     │        /   \                          /   \
     │      /       \                      /       \
     │    /           \                  /           \
  中 │  /               \              /               \
     │ /                  \          /                   \
     │/                     \      /                      \
  低 │                       \  /
     │                        \/
     └──────────────────────────────────────────────────────
       发现需求   输入主题    等待生成      收听播客     后续行为
                (期待)     (焦虑→安心)    (惊喜→满足)
```

**最大的体验风险点：** 等待生成阶段的 30-60 秒焦虑区间，是用户流失的高峰期。Terminal UI 和进度可视化是核心应对策略。
